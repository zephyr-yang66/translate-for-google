## 环境检查（强制性规则）

### 组件库依赖检查

在开始开发前，**必须**先检查项目中是否存在所需的组件库（如 PageWrapper）。

**⚠️ 重要：**
- 并非所有项目都包含 Common Admin 组件库
- 使用组件前必须先确认其存在
- 如果组件不存在，必须询问开发者替代方案

**详细的检查方法、处理流程和替代方案，请查阅：**
- [Common Admin 组件库使用文档](/docs/knowledge/frontend/knowledge-component-library.md) - "使用前必读：环境检查"章节

## 标准工作流程

```
收到任务
  ↓
0. 【强制】环境检查：确认组件库是否存在
   - 查阅组件库文档的"环境检查"章节
   - 如果组件不存在 → 立即停止并询问开发者
   - 如果存在 → 继续下一步
  ↓
1. 查看团队级规范（本文档"团队级规范"部分）
  ↓
2. 根据页面类型选择对应的开发规范
  ↓
3. 提取任务关键词（技术词 + 业务词）
  ↓
4. 使用 Grep 搜索相关规则（如需更多规则）
  ↓
5. 使用 Read tool 加载匹配的规则文件
  ↓
6. 确认上下文，明确说明已加载的规则
  ↓
7. 开始实现，遵循加载的所有规则
  ↓
8. 遇到问题？查看 FAQ 文档
```
