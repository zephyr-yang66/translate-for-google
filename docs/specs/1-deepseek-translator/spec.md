## 概述

### 功能简介
为 Google Chrome 浏览器提供一个轻量级的英译中翻译扩展，用户可以通过右键菜单快速翻译网页中的英文内容。翻译功能由 DeepSeek AI 模型提供支持，确保翻译质量自然、准确。

### 业务价值
- **提升阅读效率**：用户在浏览英文网站时无需切换到其他翻译工具，即时获取中文翻译
- **降低学习门槛**：帮助中文用户更容易理解英文技术文档、新闻、论坛等内容
- **增强用户体验**：通过侧边栏抽屉式设计，翻译结果不遮挡原文，方便对照阅读

### 目标用户
- 需要频繁阅读英文内容的中文用户
- 技术开发人员浏览英文技术文档
- 学生和研究人员查阅英文资料
- 跨境电商从业者查看英文商品描述

## 用户故事

### US-01: 快速翻译选中文本
**作为** 浏览英文网页的用户  
**我想要** 选中任意英文文本后通过右键菜单启动翻译  
**以便** 快速了解选中内容的中文含义

**验收场景**：
1. 用户在任意网页中用鼠标选中一段英文文本
2. 用户右键点击选中的文本
3. 在右键菜单中显示"DeepSeek 翻译"选项
4. 点击"DeepSeek 翻译"后，浏览器右侧弹出翻译抽屉
5. 抽屉中显示原文和对应的中文翻译结果

### US-02: 查看翻译历史
**作为** 经常使用翻译功能的用户  
**我想要** 在抽屉中看到原文和译文的对照  
**以便** 对照学习和验证翻译准确性

**验收场景**：
1. 用户通过右键菜单触发翻译
2. 翻译抽屉在浏览器窗口右侧滑出
3. 抽屉顶部显示"原文"区域，完整展示选中的英文内容
4. 抽屉中部显示"译文"区域，展示中文翻译结果
5. 原文和译文之间有明显的视觉分隔

### US-03: 关闭翻译抽屉
**作为** 已完成阅读翻译的用户  
**我想要** 方便地关闭翻译抽屉  
**以便** 恢复正常浏览并节省屏幕空间

**验收场景**：
1. 用户点击抽屉右上角的关闭按钮
2. 抽屉平滑滑出屏幕，完全消失
3. 网页内容自动回流到正常宽度
4. 用户可以再次选中文本并唤起翻译

### US-04: 配置翻译服务
**作为** 扩展的配置用户  
**我想要** 在配置页面测试和保存翻译服务设置时获得清晰的反馈  
**以便** 确认我的配置是否正确并成功保存

**验收场景**：
1. 用户在配置页面输入 API 密钥等配置信息
2. 用户点击"测试连接"按钮
3. 按钮文本变为"测试中..."并进入禁用状态，防止重复点击
4. 测试成功后显示提示："成功连接请保存"，或失败时显示："连接失败请检查"
5. 用户点击"保存"按钮
6. 保存成功后显示："保存成功，请手动关闭页面"，或失败时显示："保存失败，请重试"

### US-05: 切换翻译语言显示
**作为** 需要对照原文和译文的用户  
**我想要** 在翻译弹窗中快速切换显示的语言  
**以便** 灵活查看英文原文或中文译文

**验收场景**：
1. 用户选中网页中的文本并触发翻译
2. 翻译抽屉打开，根据选中文本的语言自动显示对应译文（中文文本显示英文，英文文本显示中文）
3. 抽屉顶部显示"中文"和"English"两个切换按钮
4. 用户点击切换按钮，内容区域切换显示原文或译文
5. 抽屉底部显示"翻译模型："下拉选择框，用户可以选择不同的翻译引擎（DeepSeek、百度翻译、LibreTranslate）

## 功能需求

### FR-01: 文本选择识别
**优先级**: P0（必须）  
**描述**: 扩展必须能够正确识别用户在网页中选中的文本内容

**详细说明**：
- 支持选中的文本类型：单词、短语、句子、段落
- 文本长度范围：1-5000 个字符
- 支持跨多个 HTML 元素的文本选择
- 自动过滤 HTML 标签，仅提取纯文本内容
- 保留原文的换行符和段落格式

**验收标准**：
- ✓ 能够正确获取单个单词（如 "hello"）
- ✓ 能够正确获取完整句子（如 "This is a sentence."）
- ✓ 能够正确获取多行段落文本
- ✓ 选中包含链接或加粗标签的文本时，仅提取文本内容，不包含标签

### FR-02: 右键菜单集成
**优先级**: P0（必须）  
**描述**: 在浏览器右键菜单中添加"DeepSeek 翻译"选项

**详细说明**：
- 仅当用户选中文本时，右键菜单才显示翻译选项
- 菜单项文本："使用 DeepSeek 翻译"
- 菜单项图标：显示扩展的 logo（可选）
- 点击菜单项后立即触发翻译流程

**验收标准**：
- ✓ 未选中文本时，右键菜单中不显示翻译选项
- ✓ 选中文本后，右键菜单中显示"使用 DeepSeek 翻译"选项
- ✓ 点击菜单项后，翻译抽屉在 500 毫秒内打开

### FR-03: 翻译抽屉 UI
**优先级**: P0（必须）  
**描述**: 在浏览器窗口右侧显示一个侧边抽屉，展示翻译内容

**详细说明**：
- 抽屉位置：浏览器窗口右侧，从右向左滑入
- 抽屉宽度：固定 400px（或占屏幕宽度的 30%，取较小值）
- 抽屉高度：占满整个浏览器窗口高度
- 抽屉背景：白色或浅灰色
- 抽屉层级：显示在网页内容之上（使用 CSS z-index）
- 关闭按钮：位于抽屉右上角，点击后关闭抽屉
- 动画效果：打开和关闭时有平滑的滑动动画（持续 300 毫秒）

**验收标准**：
- ✓ 抽屉从浏览器窗口右侧滑入
- ✓ 抽屉宽度不超过屏幕宽度的 30%，最大 400px
- ✓ 抽屉完全覆盖在网页内容之上
- ✓ 抽屉有明显的视觉边界（如阴影或边框）
- ✓ 关闭按钮清晰可见且易于点击

### FR-04: 原文展示区域
**优先级**: P0（必须）  
**描述**: 在抽屉顶部区域展示用户选中的英文原文

**详细说明**：
- 区域标题："原文" 或 "Original Text"
- 文本显示：使用等宽或易读字体
- 文本样式：灰色或深色，字号 14-16px
- 滚动支持：当原文超过显示区域时，提供垂直滚动条
- 背景色：浅灰色，与译文区域有视觉区分

**验收标准**：
- ✓ 原文完整显示，不被截断
- ✓ 原文区域最小高度 100px
- ✓ 长文本时出现滚动条
- ✓ 保留原文的换行和段落格式

### FR-05: 译文展示区域

- 翻译模型切换流畅，用户可以快速对比不同翻译引擎的结果

## 验收场景

### 场景 1：单词翻译
**步骤**：
1. 用户访问一个英文网页
2. 用户用鼠标选中单词 "documentation"
3. 用户右键点击选中的单词
4. 用户在右键菜单中点击"使用 DeepSeek 翻译"
5. 浏览器右侧滑出翻译抽屉
6. 抽屉中显示：
   - 原文："documentation"
   - 译文："文档"

**预期结果**：✅ 翻译结果正确，抽屉显示正常

---

### 场景 2：句子翻译
**步骤**：
1. 用户选中一个完整句子："This is a comprehensive guide for developers."
2. 用户右键点击并选择"使用 DeepSeek 翻译"
3. 翻译抽屉打开并显示：
   - 原文："This is a comprehensive guide for developers."
   - 译文："这是一份面向开发者的综合指南。"

**预期结果**：✅ 翻译结果准确、流畅

---

### 场景 3：多段落翻译
**步骤**：

## 假设与约束

### 假设
1. **翻译服务可用性**：假设 DeepSeek、百度翻译、LibreTranslate 提供稳定的 API 服务，并支持中英双向翻译
2. **用户环境**：假设用户使用 Google Chrome 浏览器（版本 90 或更高）
3. **网络连接**：假设用户设备有稳定的互联网连接
4. **API 密钥**：假设用户已获取相应翻译服务的 API 密钥并在配置页面中正确配置
5. **语言识别**：扩展通过字符检测（中文字符正则匹配）自动判断翻译方向

### 约束
1. **浏览器兼容性**：仅支持 Google Chrome 浏览器，不支持 Firefox、Edge 等其他浏览器
2. **翻译方向**：支持中文与英文之间的双向翻译（中译英、英译中）
3. **文本长度限制**：单次翻译的文本长度不超过 5000 个字符（受各翻译 API 限制）
4. **离线模式**：扩展需要网络连接才能工作，不支持离线翻译
5. **历史记录**：当前版本不保存翻译历史，每次新翻译会覆盖旧内容
6. **隐私要求**：选中的文本将发送到翻译服务提供商的服务器进行翻译，用户需了解并同意

### 技术约束
1. **扩展权限**：需要申请 `contextMenus`（右键菜单）、`activeTab`（访问当前标签页）权限
2. **API 配额**：受 DeepSeek API 的调用频率和配额限制
3. **跨域请求**：需要在 manifest 中配置 DeepSeek API 的域名到 `permissions` 中

## 关键实体

### 翻译请求（TranslationRequest）
**描述**：用户发起的单次翻译请求

**属性**：
- `sourceText` (String)：原文文本，1-5000 字符
- `sourceLang` (String)：源语言，"zh"（中文）或 "en"（英文），根据字符检测自动判断
- `targetLang` (String)：目标语言，"zh"（简体中文）或 "en"（英文），与 sourceLang 相反
- `timestamp` (DateTime)：请求发起时间
- `requestId` (String)：唯一请求标识符
- `translationEngine` (String)：使用的翻译引擎 - "deepseek"、"baidu"、"libretranslate"

### 翻译结果（TranslationResult）
**描述**：翻译 API 返回的翻译结果

**属性**：
- `translatedText` (String)：翻译后的文本
- `sourceText` (String)：原始文本
- `sourceLang` (String)：源语言代码
- `targetLang` (String)：目标语言代码
- `status` (Enum)：翻译状态 - `success`、`failed`、`timeout`
- `errorMessage` (String, 可选)：错误信息（仅在失败时提供）
- `responseTime` (Number)：API 响应时间（毫秒）
- `translationEngine` (String)：使用的翻译引擎

### 抽屉状态（DrawerState）
**描述**：翻译抽屉的当前状态

**属性**：
- `isOpen` (Boolean)：抽屉是否打开
- `currentRequest` (TranslationRequest, 可选)：当前正在处理的翻译请求
- `currentResult` (TranslationResult, 可选)：当前显示的翻译结果
- `isLoading` (Boolean)：是否正在加载翻译结果
- `displayLanguage` (String)：当前显示的语言 - "zh"（显示中文）或 "en"（显示英文）
- `selectedEngine` (String)：当前选中的翻译引擎 - "deepseek"、"baidu"、"libretranslate"

### 配置状态（ConfigState）
**描述**：配置页面的状态管理

**属性**：
- `isTestingConnection` (Boolean)：是否正在测试连接
- `testResult` (Object, 可选)：测试结果
  - `success` (Boolean)：测试是否成功
  - `message` (String)：提示信息 - "成功连接请保存" 或 "连接失败请检查"
- `isSaving` (Boolean)：是否正在保存配置
- `saveResult` (Object, 可选)：保存结果
  - `success` (Boolean)：保存是否成功
  - `message` (String)：提示信息 - "保存成功，请手动关闭页面" 或 "保存失败，请重试"
- `lastTestTime` (DateTime, 可选)：上次测试时间，用于防抖判断
